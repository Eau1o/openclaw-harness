# 文档花园技能

## 触发条件
- 用户说"整理文档"、"检查过时文档"、"doc-gardening"
- 每周定时任务（需配合 HEARTBEAT.md）

## 检查规则

### 规则1：文档与代码同步
扫描 docs/ 目录中引用的代码路径，检查是否存在：
```bash
# 查找文档中引用的 src/ 路径
grep -r "src/" docs/ --include="*.md" | grep -v "docs/references"
```

### 规则2：超过 90 天未更新的文档
```bash
# 查找 90 天以上未修改的 .md 文件
find docs/ -name "*.md" -type f -mtime +90
```

### 规则3：空文件或占位文档
```bash
# 查找小于 100 字节的文档
find docs/ -name "*.md" -type f -size -100c
```

### 规则4：孤立的执行计划
检查 `docs/exec-plans/active/` 中超过 7 天未更新的计划

## 输出格式

生成清理报告，格式如下：

```markdown
# 文档清理报告 - YYYY-MM-DD

## 待处理项

| 文件 | 问题类型 | 建议操作 |
|------|----------|----------|
| docs/xxx.md | 引用路径不存在 | 更新或删除引用 |
| docs/old.md | 超过90天未更新 | 审查后决定保留/归档/删除 |

## 已自动修复

（列出本次自动修复的内容）

## 需人工处理

（列出需要人工判断的内容）
```

## 自动修复能力

以下问题可自动修复：
1. 空文件 → 删除
2. 引用路径错误（简单情况）→ 尝试自动修正
3. 过时的日期标注 → 更新为当前日期
