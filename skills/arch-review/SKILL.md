# 架构审核 Skill

## 触发条件
当用户要求"审核代码"或"检查架构规范"时触发。

## 检查规则

### 规则1：分层依赖
- 检查是否存在 UI 层直接导入 Repo 层
- 检查是否存在 Service 层直接导入 UI 层
- 违规示例：`import { userRepo } from '../repo/user.repo'` 出现在 UI 层代码中

### 规则2：边界解析
- 检查外部 API 响应是否在进入 Service 层前经过 parseXxx 函数
- 检查是否有直接使用 `any` 类型接收外部数据

### 规则3：文档同步
- 检查修改的代码是否有对应的文档更新
- 检查 tech-debt-tracker.md 是否记录了临时方案

## 输出格式
对于每条违规，输出：
- ❌ 违规内容
- ✅ 修复建议
- 📖 相关文档

## 使用方法
用户说"审核这段代码"或"检查 arch"，Skill 自动执行上述规则。
